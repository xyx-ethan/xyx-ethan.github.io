    1. Game Playing

    Adversarial search for two-player, zero-sum, perfect-information games. The opponent acts rationally to minimize our utility, so we cannot simply search for a goal — we must plan against an adversary.

    1.1 Game Trees

    A game tree represents all possible move sequences. Each node is a game state; each edge is a move. Levels alternate between the two players (MAX and MIN). Terminal nodes carry utility values (+1 win, −1 loss, 0 draw, or real-valued scores).

    1.2 Minimax Algorithm

    Minimax computes the optimal strategy assuming both players play perfectly.

    • MAX nodes select the child with the highest value.
    • MIN nodes select the child with the lowest value.
    • Values propagate bottom-up from terminal states to the root.

    The minimax value of a node $n$ is:

    $$\text{minimax}(n) = \begin{cases} \text{Utility}(n) & \text{if } n \text{ is terminal} \\ \max_{a} \text{minimax}(\text{child}(n, a)) & \text{if } n \text{ is a MAX node} \\ \min_{a} \text{minimax}(\text{child}(n, a)) & \text{if } n \text{ is a MIN node} \end{cases}$$

    Complexity: Time $O(b^d)$, Space $O(bd)$ where $b$ = branching factor and $d$ = depth.

    Minimax: A decision rule for two-player zero-sum games that selects the action maximizing the minimum payoff the opponent can force. MAX chooses the move with the highest minimax value; MIN chooses the move with the lowest. Under perfect play by both sides, the minimax value of the root equals the game-theoretic value of the position.

    1.3 Alpha-Beta Pruning

    Alpha-beta pruning eliminates branches that cannot influence the final decision, without changing the result.

    • $\alpha$ = best value MAX can guarantee so far (initially $-\infty$)
    • $\beta$ = best value MIN can guarantee so far (initially $+\infty$)
    • Prune when $\alpha \geq \beta$ — the current branch is provably irrelevant.
    Alpha-beta complexity: With random move ordering, alpha-beta reduces the effective branching factor from $O(b^d)$ to $O(b^{3d/4})$. With perfect move ordering (best moves examined first), it achieves $O(b^{d/2})$ — effectively doubling the searchable depth for the same computation.
    Move ordering matters: Alpha-beta pruning only achieves its best-case $O(b^{d/2})$ complexity when the best move is examined first at each node. With poor move ordering, pruning is minimal and performance degrades toward the full $O(b^d)$ of minimax. In practice, use iterative deepening with results from the previous depth to order moves, or apply killer-move and history heuristics to approximate good ordering.

    1.4 Evaluation Functions

    Since searching to terminal states is usually infeasible, we use depth-limited search with an evaluation function at the cutoff depth. The evaluation function estimates the utility of a non-terminal state.

    • Must be fast to compute.
    • Must correlate with the true chance of winning.
    • Example for Isolation: $\text{eval}(s) = \text{#my\_moves} - \text{#opponent\_moves}$
    • More aggressive: $\text{eval}(s) = 2 \cdot \text{#my\_moves} - \text{#opponent\_moves}$

    1.5 Iterative Deepening

    Search to depth 1, then depth 2, then depth 3, etc., within a fixed time limit. When time runs out, return the best move found at the deepest completed level.

    • Always has a move ready (anytime algorithm).
    • Overhead is small because of exponential growth: the last level dominates total work. Re-doing levels 1 through $d-1$ costs roughly $\frac{1}{b-1}$ of level $d$.
    • Combines the completeness of BFS with the space efficiency of DFS.

    1.6 Quiescent Search and the Horizon Effect

    Quiescent search: At the depth limit, if the position is unstable (evaluation fluctuates significantly between consecutive levels), extend the search deeper at that node until a stable (quiescent) position is reached.

    Horizon effect: The search may push an unavoidable bad outcome beyond the depth limit by making delaying moves. The agent appears to avoid the loss, but it is merely postponing it. Quiescent search and singular extensions help mitigate this.

    2. Search

    Systematic exploration of state spaces. A search problem is defined by: initial state, actions, transition model, goal test, and path cost.

    2.1 Breadth-First Search (BFS)

    • Explores nodes level by level using a FIFO queue.
    • Complete: Yes (if branching factor $b$ is finite).
    • Optimal: Yes, when all step costs are equal.
    • Time: $O(b^d)$
    • Space: $O(b^d)$ — must store the entire frontier.

    2.2 Depth-First Search (DFS)

    • Explores the deepest node first using a LIFO stack.
    • Complete: No (can loop in infinite state spaces without cycle detection).
    • Optimal: No.
    • Time: $O(b^m)$ where $m$ is the maximum depth.
    • Space: $O(bm)$ — only stores the current path and its siblings.

    2.3 Uniform Cost Search (UCS)

    • Expands the node with the lowest path cost $g(n)$, using a priority queue.
    • Equivalent to Dijkstra's algorithm.
    • Complete: Yes (if step costs $\geq \epsilon > 0$).
    • Optimal: Yes, for non-negative edge costs.
    • Time & Space: $O(b^{1 + \lfloor C^*/\epsilon \rfloor})$ where $C^*$ is the optimal cost.

    2.4 A* Search

    A* uses the evaluation function:

    $$f(n) = g(n) + h(n)$$

    where $g(n)$ is the cost from start to $n$, and $h(n)$ is a heuristic estimate of cost from $n$ to the goal.

    • Admissibility: $h(n) \leq h^*(n)$ for all $n$, where $h^*(n)$ is the true cost. Guarantees optimality for tree search.
    • Consistency (monotonicity): $h(n) \leq c(n, n') + h(n')$ for every successor $n'$. Guarantees optimality for graph search.
    • Consistency implies admissibility (but not vice versa).
    A* Search: A best-first graph search algorithm that expands nodes in order of $f(n) = g(n) + h(n)$, where $g(n)$ is the actual cost from the start and $h(n)$ is a heuristic estimate of the remaining cost to the goal. A* is both complete and optimal provided the heuristic is admissible (for tree search) or consistent (for graph search). It is optimally efficient among all algorithms that use the same heuristic information.
    Admissible heuristic: A heuristic $h(n)$ is admissible if it never overestimates the true cost to the goal: $h(n) \leq h^*(n)$ for all $n$. Admissibility guarantees that A* tree search finds the optimal solution because no promising node is skipped. Common admissible heuristics include straight-line (Euclidean) distance and Manhattan distance on grids. A stronger condition, consistency ($h(n) \leq c(n, n') + h(n')$), is needed for optimality in graph search.
    A* optimality: A* with a consistent heuristic expands nodes in non-decreasing order of $f$ values. Any node on the optimal path has $f(n) = g(n) + h(n) \leq g(n) + h^*(n) = C^*$, so it will be expanded before any suboptimal goal node $n'$ with $f(n') = g(n') > C^*$.

    2.5 Bidirectional Search

    Run two simultaneous searches: forward from the start and backward from the goal. They meet in the middle.

    • Time & Space: $O(b^{d/2})$ — exponentially better than unidirectional search.
    • Requires that predecessors of a state be efficiently computable.

    3. Simulated Annealing

    Optimization techniques that escape local optima through controlled randomness. Applicable when the state space is too large for systematic search.

    3.1 Hill Climbing and Its Limitations

    Hill climbing (greedy local search) repeatedly moves to the neighbor with the highest value. It is fast but suffers from:

    • Local optima: peaks that are not the global maximum.
    • Ridges: narrow elevated regions that are hard to traverse.
    • Plateaus: flat regions with no uphill direction; includes shoulders (eventually lead somewhere) and flat maxima.

    3.2 Simulated Annealing (SA)

    Inspired by the metallurgical process of annealing. SA allows downhill moves with a probability that decreases over time.

    For a proposed move with energy change $\Delta E$:

    • If $\Delta E > 0$ (improving), always accept.
    • If $\Delta E \leq 0$ (worsening), accept with probability:
    $$P(\text{accept}) = e^{\Delta E / T}$$

    where $T$ is the current temperature.

    • High $T$: almost all moves accepted (random walk / exploration).
    • Low $T$: only improving moves accepted (hill climbing / exploitation).
    • As $T \to 0$, SA converges to hill climbing.
    Simulated Annealing: A probabilistic local search algorithm that escapes local optima by accepting worse solutions with a probability $P = e^{\Delta E / T}$ that decreases as the temperature $T$ is lowered over time. At high temperatures the search behaves like a random walk (exploration); at low temperatures it behaves like hill climbing (exploitation). With a sufficiently slow cooling schedule, SA converges to the global optimum with probability 1.
    SA convergence: If the cooling schedule decreases $T$ slowly enough (e.g., $T(t) = T_0 / \log(1+t)$), SA will find the global optimum with probability approaching 1. In practice, faster geometric schedules like $T(t) = T_0 \cdot \alpha^t$ with $0.9 \leq \alpha < 1$ are used.

    3.3 Cooling Schedule

    The cooling schedule determines how $T$ decreases over time and controls the exploration-exploitation tradeoff:

    • Too fast: gets stuck in local optima (insufficient exploration).
    • Too slow: wastes computation (too much random wandering).
    • Common schedule: geometric decay $T_{k+1} = \alpha \cdot T_k$ with $\alpha \in [0.90, 0.99]$.

    3.4 Genetic Algorithms

    Population-based search inspired by natural selection:

    1. Selection: Choose parents based on fitness (e.g., roulette wheel, tournament selection).
    2. Crossover: Combine parts of two parents to produce offspring (single-point, two-point, or uniform crossover).
    3. Mutation: Randomly alter offspring with small probability to maintain diversity.

    The population evolves over generations, preserving and combining good building blocks (schema theory).

    4. Constraint Satisfaction

    CSPs model problems through variables, domains, and constraints rather than explicit state-space search. The goal is to find an assignment of values to variables that satisfies all constraints simultaneously.

    4.1 CSP Formulation

    A CSP is defined by the triple $(X, D, C)$:

    • $X = \{X_1, X_2, \dots, X_n\}$: a set of variables.
    • $D = \{D_1, D_2, \dots, D_n\}$: a set of domains (possible values for each variable).
    • $C$: a set of constraints specifying allowed combinations of values.

    Example — Map Coloring: Variables = regions, Domains = {Red, Green, Blue}, Constraints = adjacent regions must have different colors.

    Constraint Satisfaction Problem (CSP): A problem defined by a set of variables $X$, a set of domains $D$ (one per variable), and a set of constraints $C$ that specify which combinations of values are allowed. A solution is a complete assignment of values to all variables such that every constraint is satisfied. CSPs abstract away the notion of path cost and focus purely on feasibility.
    Map coloring: Consider coloring the map of Australia with 3 colors (Red, Green, Blue). The 7 states/territories are variables, each with domain $\{R, G, B\}$. Adjacent regions must differ in color, yielding constraints like $\text{WA} \neq \text{NT}$, $\text{NT} \neq \text{SA}$, etc. Backtracking with MRV picks SA first (most neighbors = 5), assigns it Red, then propagates constraints via forward checking. With AC-3, the problem can often be solved with zero backtracks.

    4.2 Backtracking Search

    Depth-first search with one variable assigned per level. If a partial assignment violates a constraint, backtrack immediately (prune).

    • Assign variables one at a time (not all at once).
    • Check constraints as soon as a variable is assigned.
    • Worst case: $O(d^n)$ where $d$ = domain size, $n$ = number of variables.

    4.3 Arc Consistency (AC-3)

    A variable $X_i$ is arc-consistent with $X_j$ if for every value in $D_i$, there exists some value in $D_j$ that satisfies the constraint between them.

    AC-3 algorithm:

    1. Initialize a queue with all arcs $(X_i, X_j)$.
    2. For each arc, remove values from $D_i$ that have no consistent partner in $D_j$.
    3. If $D_i$ changed, re-add all arcs $(X_k, X_i)$ to the queue.
    4. If any domain becomes empty, the CSP is unsolvable.

    Complexity: $O(ed^3)$ where $e$ = number of arcs, $d$ = max domain size.

    4.4 Forward Checking

    After assigning a value to $X_i$, immediately eliminate inconsistent values from the domains of all unassigned neighbors. If any neighbor's domain becomes empty, backtrack.

    Forward checking is weaker than full arc consistency but cheaper per step.

    4.5 Variable and Value Ordering Heuristics

    • MRV (Minimum Remaining Values): Choose the variable with the fewest legal values remaining. Also called fail-first — it detects failures early, reducing the search tree.
    • Degree heuristic: Tie-breaker for MRV. Choose the variable involved in the most constraints with unassigned variables.
    • LCV (Least Constraining Value): Choose the value that rules out the fewest values in neighboring domains. This maximizes flexibility for future assignments.
    MRV vs. LCV: MRV is for variable ordering (fail-first: try the hardest variable next to detect dead ends early). LCV is for value ordering (fail-last: try the least restrictive value first to maximize chance of success).

    5. Probability

    The mathematical framework for reasoning under uncertainty. AI agents must handle incomplete information and noisy observations.

    5.1 Probability Axioms

    For a sample space $\Omega$ and events $A, B \subseteq \Omega$:

    1. $0 \leq P(A) \leq 1$ for all events $A$.
    2. $P(\Omega) = 1$.
    3. Additivity: If $A$ and $B$ are mutually exclusive ($A \cap B = \emptyset$), then $P(A \cup B) = P(A) + P(B)$.

    Useful consequences:

    • $P(\neg A) = 1 - P(A)$
    • $P(A \cup B) = P(A) + P(B) - P(A \cap B)$ (inclusion-exclusion)

    5.2 Conditional Probability

    The probability of $A$ given that $B$ is known to have occurred:

    $$P(A \mid B) = \frac{P(A \cap B)}{P(B)}, \quad P(B) > 0$$

    This is the primary mechanism for updating beliefs given new evidence.

    From this definition follows the product rule:

    $$P(A \cap B) = P(A \mid B) \cdot P(B) = P(B \mid A) \cdot P(A)$$

    5.3 Law of Total Probability

    If $\{A_1, A_2, \dots, A_n\}$ is a partition of $\Omega$ (mutually exclusive, collectively exhaustive), then:

    $$P(B) = \sum_{i=1}^{n} P(B \mid A_i) \, P(A_i)$$

    This allows computing $P(B)$ by summing over all possible "causes" $A_i$.

    5.4 Bayes' Rule

    Bayes' rule relates $P(\text{cause} \mid \text{effect})$ to $P(\text{effect} \mid \text{cause})$:

    $$P(A \mid B) = \frac{P(B \mid A) \, P(A)}{P(B)}$$

    In terms of named components:

    $$\underbrace{P(A \mid B)}_{\text{posterior}} = \frac{\overbrace{P(B \mid A)}^{\text{likelihood}} \cdot \overbrace{P(A)}^{\text{prior}}}{\underbrace{P(B)}_{\text{evidence}}}$$
    Bayes' Theorem: A formula for inverting conditional probabilities: $P(A \mid B) = P(B \mid A) \, P(A) \;/\; P(B)$. It updates a prior belief $P(A)$ using the likelihood $P(B \mid A)$ of observed evidence to produce a posterior belief $P(A \mid B)$. The denominator $P(B)$ is a normalizing constant computed via the law of total probability. Bayes' theorem is the foundation of all probabilistic inference in AI.
    Diagnostic vs. causal reasoning: Bayes' rule lets us invert directions. If we know $P(\text{symptom} \mid \text{disease})$ (causal, easy to estimate), we can compute $P(\text{disease} \mid \text{symptom})$ (diagnostic, what we need).
    Medical test (base rate fallacy): A disease affects 1 in 1000 people. A test has 99% sensitivity ($P(+ \mid \text{sick}) = 0.99$) and 1% false positive rate ($P(+ \mid \text{healthy}) = 0.01$). If you test positive, what is $P(\text{sick} \mid +)$? By Bayes' rule: $P(\text{sick} \mid +) = \frac{0.99 \times 0.001}{0.99 \times 0.001 + 0.01 \times 0.999} \approx 9\%$. Despite the accurate test, the low base rate means most positives are false positives.

    5.5 Independence and Conditional Independence

    Independence: $A$ and $B$ are independent if:

    $$P(A \cap B) = P(A) \cdot P(B) \quad \Leftrightarrow \quad P(A \mid B) = P(A)$$

    Conditional independence: $A$ and $B$ are conditionally independent given $C$ if:

    $$P(A \cap B \mid C) = P(A \mid C) \cdot P(B \mid C)$$

    Conditional independence is the key to compact probabilistic models (Bayes nets). Two variables may be marginally dependent but conditionally independent (or vice versa).

    6. Bayes Nets

    Bayesian networks are directed acyclic graphs (DAGs) that compactly represent a joint probability distribution using conditional independence relationships.

    6.1 Structure and Semantics

    A Bayes net consists of:

    • Nodes: random variables.
    • Edges: direct probabilistic dependencies (parent → child).
    • CPTs (Conditional Probability Tables): each node stores $P(X_i \mid \text{Parents}(X_i))$.
    Bayesian Network: A directed acyclic graph (DAG) where each node represents a random variable and each directed edge encodes a direct probabilistic dependency. Each node stores a conditional probability table (CPT) $P(X_i \mid \text{Parents}(X_i))$. The full joint distribution factorizes as $P(X_1, \ldots, X_n) = \prod_i P(X_i \mid \text{Parents}(X_i))$, which is exponentially more compact than a flat joint table when each variable has few parents.

    The joint distribution factorizes as:

    $$P(X_1, X_2, \dots, X_n) = \prod_{i=1}^{n} P(X_i \mid \text{Parents}(X_i))$$

    Compactness: Instead of $O(2^n)$ entries for the full joint table, a Bayes net with $n$ Boolean variables and at most $k$ parents per node requires $O(n \cdot 2^k)$ parameters.

    6.2 D-Separation

    D-separation determines conditional independence directly from the graph structure. There are three canonical cases:

    StructureNameFlow without evidenceFlow given evidence on B
    $A \to B \to C$ChainYes (A and C dependent)Blocked (A ⊥ C | B)
    $A \leftarrow B \to C$Fork (common cause)Yes (A and C dependent)Blocked (A ⊥ C | B)
    $A \to B \leftarrow C$Collider (common effect)Blocked (A ⊥ C)Unblocked! (A and C become dependent)
    Explaining away (collider): In a collider $A \to B \leftarrow C$, $A$ and $C$ are marginally independent. But once we observe $B$ (or any descendant of $B$), knowing about $A$ changes our belief about $C$. Example: two independent causes of a car not starting — once we know it won't start, ruling out one cause makes the other more likely.

    General d-separation rule: $X$ and $Y$ are d-separated given $Z$ if every undirected path between $X$ and $Y$ is blocked. A path is blocked if it contains a node $V$ such that either: (a) $V \in Z$ and $V$ is in a chain or fork, or (b) $V \notin Z$ and $V$ is a collider (and no descendant of $V$ is in $Z$).

    6.3 Inference by Enumeration

    To compute $P(Q \mid e)$, sum over all values of hidden variables $H$:

    $$P(Q \mid e) = \frac{P(Q, e)}{P(e)} = \frac{\sum_{h} P(Q, e, h)}{\sum_{q} \sum_{h} P(q, e, h)}$$

    This is exact but exponential in the number of hidden variables.

    6.4 Variable Elimination

    Optimizes enumeration by factoring out variables one at a time. Instead of summing over the full joint, we:

    1. Express the joint as a product of factors (CPTs).
    2. Eliminate (sum out) hidden variables one at a time, creating intermediate factors.
    3. The elimination order affects efficiency — choosing a good order is NP-hard in general, but heuristics (min-neighbors, min-fill) work well in practice.

    6.5 Approximate Inference: Sampling Methods

    For large networks, exact inference is intractable. Sampling methods provide approximate answers:

    • Rejection sampling: Generate samples from the prior, reject those inconsistent with evidence. Simple but wasteful — if evidence is unlikely, most samples are rejected.
    • Likelihood weighting: Fix evidence variables and weight each sample by $\prod P(e_i \mid \text{parents}(e_i))$. All samples are consistent with evidence, so none are wasted. But downstream evidence is not handled well.
    • Gibbs sampling (MCMC): Start with a random assignment. Repeatedly pick a non-evidence variable and resample it given its Markov blanket (parents, children, and children's other parents). Converges to the true posterior over time.

    6.6 Naive Bayes Classifier

    Assumes all features $F_1, \dots, F_n$ are conditionally independent given the class $C$:

    $$P(C \mid F_1, \dots, F_n) \propto P(C) \prod_{i=1}^{n} P(F_i \mid C)$$

    Despite the strong independence assumption, Naive Bayes works well in many practical classification tasks (spam detection, text classification).

    7. Machine Learning

    Learning patterns from data. Supervised learning: given labeled examples $(x, y)$, learn $f: X \to Y$. Classification (discrete $Y$) vs. regression (continuous $Y$).

    7.1 k-Nearest Neighbors (k-NN)

    k-Nearest Neighbors (k-NN): A non-parametric, instance-based learning algorithm that classifies a query point by majority vote among its $k$ closest training examples under a chosen distance metric. The training data itself serves as the model, requiring no explicit training phase. At prediction time the algorithm computes distances to all training points, making it $O(n)$ per query without indexing structures such as KD-trees.
    • Classify a query point by majority vote of its $k$ nearest training examples.
    • Non-parametric: no explicit model; the training data is the model.
    • Sensitive to the choice of distance metric (Euclidean, Manhattan, etc.) and the value of $k$.
    • Small $k$: low bias, high variance (overfitting). Large $k$: high bias, low variance (underfitting).
    • Curse of dimensionality: in high dimensions, all points become nearly equidistant.

    7.2 Decision Trees

    Recursively partition the feature space by selecting the feature that provides the most information.

    Entropy of a set $S$ with $c$ classes:

    $$H(S) = -\sum_{i=1}^{c} p_i \log_2 p_i$$

    where $p_i$ is the proportion of class $i$ in $S$. Entropy is maximized when all classes are equally likely and zero when the set is pure (all one class).

    Information Gain for splitting on attribute $A$:

    $$\text{IG}(S, A) = H(S) - \sum_{v \in \text{Values}(A)} \frac{|S_v|}{|S|} H(S_v)$$

    Choose the attribute with the highest information gain at each node. Stop when a node is pure or no further splits improve the gain.

    • Pruning: Remove branches that do not improve validation accuracy to prevent overfitting.
    • Gain ratio: Normalizes information gain by split information to avoid bias toward many-valued attributes.

    7.3 Neural Networks

    Layers of interconnected nodes with trainable weights. Each node computes:

    $$z = \sum_i w_i x_i + b, \quad a = \sigma(z)$$

    Common activation functions:

    • Sigmoid: $\sigma(z) = \frac{1}{1 + e^{-z}}$, output in $(0, 1)$.
    • ReLU: $\sigma(z) = \max(0, z)$, avoids vanishing gradient problem.

    Backpropagation: Compute the gradient of the loss function with respect to each weight by applying the chain rule layer by layer from output to input. Use gradient descent to update weights:

    $$w \leftarrow w - \eta \frac{\partial \mathcal{L}}{\partial w}$$

    where $\eta$ is the learning rate.

    7.4 Support Vector Machines (SVMs)

    Find the maximum-margin hyperplane separating two classes. The margin is the distance between the hyperplane and the nearest data points (support vectors).

    For a hyperplane $\mathbf{w} \cdot \mathbf{x} + b = 0$, the margin is $\frac{2}{\|\mathbf{w}\|}$. The optimization problem:

    $$\min_{\mathbf{w}, b} \frac{1}{2}\|\mathbf{w}\|^2 \quad \text{s.t.} \quad y_i(\mathbf{w} \cdot \mathbf{x}_i + b) \geq 1 \;\; \forall i$$

    Kernel trick: Map data to a higher-dimensional space via $\phi(\mathbf{x})$ without computing $\phi$ explicitly. The kernel $K(\mathbf{x}_i, \mathbf{x}_j) = \phi(\mathbf{x}_i) \cdot \phi(\mathbf{x}_j)$ enables non-linear decision boundaries. Common kernels: polynomial, RBF (Gaussian).

    7.5 Boosting (AdaBoost)

    Combine multiple weak learners (slightly better than random) into a strong learner:

    1. Train a weak learner on the weighted training data.
    2. Increase weights on misclassified examples.
    3. Train the next weak learner on the re-weighted data.
    4. Final prediction: weighted majority vote of all weak learners.

    The weight of learner $t$ is: $\alpha_t = \frac{1}{2} \ln\!\left(\frac{1 - \epsilon_t}{\epsilon_t}\right)$ where $\epsilon_t$ is its weighted error rate.

    7.6 Bias-Variance Tradeoff

    For a model's expected prediction error:

    $$\text{Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Noise}$$
    • High bias (underfitting): model is too simple, misses real patterns.
    • High variance (overfitting): model is too complex, fits noise in training data.

    Cross-validation: Split data into $k$ folds; train on $k-1$ folds and validate on the remaining one; rotate through all $k$ folds. Provides a robust estimate of generalization error for model selection.

    Overfitting vs. underfitting: A model that memorizes the training data (e.g., $k = 1$ in k-NN, an unpruned decision tree) achieves near-zero training error but generalizes poorly — this is overfitting. Conversely, a model that is too simple (e.g., $k = n$ in k-NN, a shallow tree) fails to capture real patterns — this is underfitting. Always evaluate on held-out data using cross-validation, and tune model complexity to minimize validation error, not training error.

    8. Pattern Recognition Through Time

    Hidden Markov Models (HMMs) for modeling sequential/temporal data where the underlying process is not directly observable.

    8.1 Markov Chains

    A sequence of random variables $X_1, X_2, \dots$ satisfying the Markov property:

    $$P(X_t \mid X_{t-1}, X_{t-2}, \dots, X_1) = P(X_t \mid X_{t-1})$$

    The future depends only on the present, not the past. Fully characterized by transition probabilities $a_{ij} = P(X_t = j \mid X_{t-1} = i)$.

    8.2 Hidden Markov Model Definition

    Hidden Markov Model (HMM): A generative probabilistic model for sequential data defined by the tuple $\lambda = (A, B, \pi)$: a state transition matrix $A$, an emission (observation) matrix $B$, and an initial state distribution $\pi$. The system transitions between hidden states according to a first-order Markov chain, and each hidden state emits an observable output. The three core problems — evaluation (Forward), decoding (Viterbi), and learning (Baum-Welch) — are each solved in $O(N^2 T)$ time.

    An HMM is defined by the tuple $\lambda = (A, B, \pi)$:

    • $A = [a_{ij}]$: transition matrix — $a_{ij} = P(q_{t+1} = S_j \mid q_t = S_i)$.
    • $B = [b_j(o_k)]$: emission (observation) matrix — $b_j(o_k) = P(O_t = o_k \mid q_t = S_j)$.
    • $\pi = [\pi_i]$: initial state distribution — $\pi_i = P(q_1 = S_i)$.

    At each time step, the system is in a hidden state $q_t$ and emits an observable output $O_t$.

    8.3 Three Fundamental Problems

    Problem 1 — Evaluation: Given model $\lambda$ and observation sequence $O = O_1, \dots, O_T$, compute $P(O \mid \lambda)$.

    Solved by the Forward algorithm. Define the forward variable:

    $$\alpha_t(j) = P(O_1, O_2, \dots, O_t, \; q_t = S_j \mid \lambda)$$

    Recurrence:

    • Initialization: $\alpha_1(j) = \pi_j \cdot b_j(O_1)$
    • Induction:
    $$\alpha_t(j) = \left[\sum_{i=1}^{N} \alpha_{t-1}(i) \cdot a_{ij}\right] \cdot b_j(O_t)$$
    • Termination: $P(O \mid \lambda) = \sum_{i=1}^{N} \alpha_T(i)$

    Complexity: $O(N^2 T)$ instead of naive $O(N^T)$, where $N$ = number of states, $T$ = sequence length.

    Forward algorithm intuition: $\alpha_t(j)$ is the probability of having seen observations $O_1, \dots, O_t$ and being in state $S_j$ at time $t$. We build this up incrementally: at each step, sum over all ways to arrive at $S_j$ from any state $S_i$ at $t-1$, weighted by the transition probability $a_{ij}$, then multiply by the emission probability $b_j(O_t)$.

    Problem 2 — Decoding: Find the most likely state sequence $Q^* = \arg\max_Q P(Q \mid O, \lambda)$.

    Solved by the Viterbi algorithm. Same structure as Forward, but replaces sum with max:

    $$\delta_t(j) = \max_{q_1, \dots, q_{t-1}} P(q_1, \dots, q_{t-1}, q_t = S_j, O_1, \dots, O_t \mid \lambda)$$

    Recurrence:

    $$\delta_t(j) = \max_{i} \left[\delta_{t-1}(i) \cdot a_{ij}\right] \cdot b_j(O_t)$$

    Maintain backpointers $\psi_t(j) = \arg\max_i [\delta_{t-1}(i) \cdot a_{ij}]$ to trace back the optimal state sequence.

    Problem 3 — Learning: Adjust $\lambda = (A, B, \pi)$ to maximize $P(O \mid \lambda)$.

    Solved by the Baum-Welch algorithm (a special case of EM):

    1. E-step: Compute expected state occupancies and transitions using Forward-Backward.
    2. M-step: Re-estimate $A$, $B$, $\pi$ from the expected counts.
    3. Repeat until convergence (guaranteed to reach a local maximum of likelihood).

    9. Logic and Planning

    Formal reasoning systems and automated planning for deterministic environments.

    9.1 Propositional Logic

    Propositions are statements that are either true or false. Combined with logical connectives:

    ConnectiveSymbolMeaning
    Negation$\neg P$NOT $P$
    Conjunction$P \land Q$$P$ AND $Q$
    Disjunction$P \lor Q$$P$ OR $Q$
    Implication$P \Rightarrow Q$If $P$ then $Q$
    Biconditional$P \Leftrightarrow Q$$P$ if and only if $Q$

    Key inference rules:

    • Modus Ponens: From $P$ and $P \Rightarrow Q$, infer $Q$.
    • Resolution: From $P \lor Q$ and $\neg P \lor R$, infer $Q \lor R$. Basis for efficient automated theorem proving.
    Resolution Refutation: A proof technique that establishes a statement $\alpha$ by showing that $\neg \alpha$ leads to a contradiction. Convert the knowledge base and $\neg \alpha$ to conjunctive normal form (CNF), then repeatedly apply the resolution rule — resolving complementary literals across clauses — until the empty clause (a contradiction) is derived. If the empty clause is produced, $\alpha$ is proven; if no new resolvents can be generated, $\alpha$ cannot be proven from the knowledge base.

    9.2 First-Order Logic (FOL)

    Extends propositional logic with variables, predicates, functions, and quantifiers:

    • Predicates: $\text{OnTable}(x)$, $\text{Loves}(x, y)$ — express properties and relations.
    • Universal quantifier: $\forall x \; P(x)$ — "for all $x$, $P(x)$ holds."
    • Existential quantifier: $\exists x \; P(x)$ — "there exists an $x$ such that $P(x)$ holds."

    Unification: Finding a substitution $\theta$ that makes two expressions identical. Example: Unify $\text{Knows}(John, x)$ with $\text{Knows}(y, Jane)$ yields $\theta = \{y/John, x/Jane\}$. Unification is essential for applying inference rules in FOL.

    9.3 Classical Planning (STRIPS)

    STRIPS (Stanford Research Institute Problem Solver) represents planning problems with:

    • States: Sets of ground atomic propositions (e.g., $\{\text{On}(A,B), \text{Clear}(A), \text{OnTable}(B)\}$).
    • Actions: Defined by:
      • Preconditions: propositions that must be true for the action to apply.
      • Effects: propositions added (add list) and removed (delete list) after the action.
    • Goal: Find a sequence of actions transforming the initial state into a state satisfying the goal conditions.

    9.4 Situation Calculus

    Represents dynamic worlds in first-order logic. A situation is a history of actions. Key elements:

    • Fluents: Predicates whose truth varies across situations, e.g., $\text{On}(A, B, s)$.
    • Successor-state axioms: Describe how fluents change. Address the frame problem by specifying what doesn't change.

    9.5 GraphPlan

    Builds a layered planning graph alternating between proposition layers and action layers:

    1. Start with the initial state propositions.
    2. Add all applicable actions and their effects.
    3. Identify mutex (mutual exclusion) relations between actions and propositions.
    4. Expand until all goal propositions appear in the same layer without mutex conflicts.
    5. Extract a valid plan by backward search through the graph.

    The planning graph provides a powerful heuristic for the reachability of goals and is the basis for the level-based heuristic used by many modern planners.

    10. Planning Under Uncertainty

    Decision making when actions have uncertain outcomes. Markov Decision Processes provide the mathematical framework for sequential decision making under uncertainty.

    10.1 MDP Definition

    Markov Decision Process (MDP): A mathematical framework for sequential decision making under uncertainty, defined by the tuple $(S, A, T, R, \gamma)$: states, actions, a transition function $T(s, a, s') = P(s' \mid s, a)$, a reward function $R(s, a)$, and a discount factor $\gamma \in [0, 1)$. The Markov property ensures the next state depends only on the current state and action, not on history. The goal is to find a policy $\pi^*$ that maximizes expected cumulative discounted reward.

    An MDP is defined by the tuple $(S, A, T, R, \gamma)$:

    • $S$: finite set of states.
    • $A$: finite set of actions.
    • $T(s, a, s') = P(s' \mid s, a)$: transition function — probability of reaching $s'$ from $s$ via action $a$.
    • $R(s, a)$: reward function — immediate reward for taking action $a$ in state $s$.
    • $\gamma \in [0, 1)$: discount factor.

    A policy $\pi: S \to A$ maps states to actions. The optimal policy $\pi^*$ maximizes expected cumulative discounted reward.

    10.2 Bellman Equation

    The optimal value function $V^*(s)$ satisfies the Bellman optimality equation:

    $$V^*(s) = \max_{a} \left[ R(s, a) + \gamma \sum_{s'} T(s, a, s') \, V^*(s') \right]$$

    The optimal policy is then:

    $$\pi^*(s) = \arg\max_{a} \left[ R(s, a) + \gamma \sum_{s'} T(s, a, s') \, V^*(s') \right]$$
    Bellman equation as a fixed point: The Bellman optimality equation expresses $V^*(s)$ recursively in terms of itself: the value of a state equals the best immediate reward plus the discounted expected value of the next state. This self-consistency condition has a unique solution, and both value iteration (repeated application of the Bellman backup operator) and policy iteration converge to it. The discount factor $\gamma < 1$ makes the operator a contraction, guaranteeing convergence.

    10.3 Value Iteration

    Iteratively update the value function until convergence:

    $$V_{k+1}(s) = \max_{a} \left[ R(s, a) + \gamma \sum_{s'} T(s, a, s') \, V_k(s') \right]$$
    • Initialize $V_0(s) = 0$ for all $s$ (or any values).
    • Repeat until $\max_s |V_{k+1}(s) - V_k(s)| < \epsilon$.
    • Convergence: Guaranteed because the Bellman operator is a contraction mapping with factor $\gamma$.
    • Complexity per iteration: $O(|S|^2 |A|)$.

    10.4 Policy Iteration

    Alternates between two steps:

    1. Policy Evaluation: Given policy $\pi$, solve for $V^\pi(s)$: $$V^\pi(s) = R(s, \pi(s)) + \gamma \sum_{s'} T(s, \pi(s), s') \, V^\pi(s')$$ This is a system of $|S|$ linear equations in $|S|$ unknowns — solvable directly or iteratively.
    2. Policy Improvement: Update the policy greedily: $$\pi'(s) = \arg\max_{a} \left[ R(s, a) + \gamma \sum_{s'} T(s, a, s') \, V^\pi(s') \right]$$

    Repeat until the policy no longer changes. Converges in a finite number of iterations (the number of deterministic policies is finite).

    Value iteration vs. policy iteration: Value iteration performs many small updates; convergence can be slow but each step is cheap. Policy iteration converges in fewer outer iterations, but each policy evaluation step can be expensive (solving a linear system). In practice, policy iteration often converges faster for small to moderate state spaces.

    10.5 Discount Factor $\gamma$

    • $\gamma$ close to 0: agent is myopic, cares only about immediate reward.
    • $\gamma$ close to 1: agent is far-sighted, values future rewards almost as much as immediate ones.
    • $\gamma < 1$ ensures the infinite sum of rewards converges and prevents infinite utility in cyclic environments.
    • The effective planning horizon is roughly $\frac{1}{1 - \gamma}$ steps.

    10.6 POMDPs: Partial Observability

    In a Partially Observable MDP, the agent cannot directly observe the state. Instead:

    • The agent maintains a belief state $b(s)$ — a probability distribution over states.
    • After taking action $a$ and receiving observation $o$, the belief is updated via Bayes' rule.
    • The POMDP can be converted to a continuous-state MDP over belief space, but this is computationally much harder than a standard MDP.

    10.7 Reinforcement Learning: Q-Learning

    When the transition function $T$ and reward function $R$ are unknown, the agent must learn from experience. Q-learning learns the optimal action-value function directly:

    $$Q(s, a) \leftarrow Q(s, a) + \alpha \left[ r + \gamma \max_{a'} Q(s', a') - Q(s, a) \right]$$

    where:

    • $\alpha$: learning rate (step size).
    • $r$: observed reward after taking action $a$ in state $s$.
    • $s'$: resulting state.
    • $\gamma \max_{a'} Q(s', a')$: estimated optimal future value.
    Q-learning properties: Q-learning is model-free (doesn't need $T$ or $R$) and off-policy (learns the optimal policy regardless of the exploration policy used, as long as all state-action pairs are visited infinitely often). The exploration-exploitation dilemma is typically managed with an $\epsilon$-greedy strategy.
